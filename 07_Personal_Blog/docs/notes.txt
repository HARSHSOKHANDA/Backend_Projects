I’ll explain what each package does and why you need it in a blog project—no fluff.
✅ 1. express
What it is:
A Node.js web framework.

Why you need it:
It handles routes, requests, responses, middleware.
Without Express, building a backend is painful.
👉 Example:
/login, /register, /posts, /api/blogs

✅ 2. mongoose
What it is:
A MongoDB ODM (Object Data Modeling).

Why you need it:
To connect MongoDB and define schemas/models.

👉 Example:
User model, Blog model, Comment model.

✅ 3. bcrypt
What it is:
Password hashing library.

Why you need it:
To encrypt passwords before saving in database.
If DB is hacked, real passwords are safe.

❌ Storing plain passwords = TRASH security

✅ 4. jsonwebtoken (JWT)
What it is:
Creates and verifies authentication tokens.

Why you need it:
To keep users logged in without sessions (for APIs).
Used in MERN apps, mobile apps, SPA apps.

👉 Example:
User logs in → server sends JWT → browser sends token in requests.

✅ 5. express-session
What it is:
Session middleware for Express.

Why you need it:
To store user login session on server.
Alternative to JWT.
👉 Example:
After login, server remembers user with session ID.

✅ 6. connect-mongo
What it is:
Stores sessions in MongoDB.

Why you need it:
By default sessions store in RAM (bad for production).
This saves sessions in database.
👉 Needed for scalable apps.

✅ 7. cookie-parser
What it is:
Parses cookies from browser.

Why you need it:
Sessions and JWT often use cookies.
Without this, Express can’t read cookies easily.

✅ 8. dotenv
What it is:
Loads environment variables from .env file.

Why you need it:
To hide secrets like:
Database URL
JWT secret
Session secret

❌ Never hardcode secrets in code (rookie mistake)

✅ 9. ejs
What it is:
Template engine (HTML with JS).

Why you need it:
To render dynamic pages like:
Blog list
Single blog page
Dashboard
👉 Server-side rendering (not React).

✅ 10. express-ejs-layouts
What it is:
Layout support for EJS.
Why you need it:
To reuse header/footer/navbar across pages.
Instead of repeating HTML in every file.

✅ 11. method-override (you typed mathod-override)
What it is:
Allows PUT & DELETE requests from HTML forms.

Why you need it:
HTML forms only support:
GET
POST
But CRUD needs:
PUT
DELETE
This package fakes those methods.

## What is JWT?
JWT stands for JSON Web Token. JWT is a self-contained, digitally signed token that allows the server to verify the identity 
of a user on every request without querying the database — making authentication fast, stateless, and scalable

## Why use it?
HTTP is stateless — the server forgets who you are after every request. JWT solves this problem.

Real world example:
"Think of JWT like a passport. At the airport, you prove your identity once, get a passport stamped, 
and then show that passport everywhere — without proving your citizenship again and again. JWT works the same way."

## JWT vs Session — why prefer JWT?
Both do the same job BUT differently:
SESSION:                          JWT:
    ↓                               ↓
Server memory/DB में               Token के अंदर
data save होता है                  data save होता है
    ↓                               ↓
Cookie में sirf                    Cookie में poora
Session ID जाता है                 Token जाता है
    ↓                               ↓
Har request pe                     Har request pe
DB check karo                      Sirf verify karo

--> Interview answer:
"Session and JWT both solve statelessness but differently. 
Session stores data on the server and sends only an ID to the browser — requiring a database lookup on every request. 
JWT stores data inside the token itself — so the server just verifies the signature without touching the database. 
In my project, I use Session for flash messages and JWT for authentication because they each excel at different things."

____________________________ Explaination of the Project ____________________________________
I built a backend for a personal blog application using Node.js, Express, and MongoDB.
The project allows an admin to log in securely, create, edit, and delete blog posts,
and users can view posts with pagination and search functionality.
I designed the project using an MVC architecture with an additional service layer.
Routes handle incoming requests, controllers manage requests and responses, and services
contain business logic and database operations. This separation helps keep the code clean,
scalable, and reusable. For authentication, I used JWT stored in HTTP-only cookies and
protected admin routes using middleware. I also implemented pagination on the homepage and
a search feature using MongoDB queries for better performance and usability.
Overall, my focus was not just functionality but also clean architecture, so I structured the
project with controllers, services, middlewares, and models to follow industry practices.
